# Product Demo Content Pattern

product-demo 的视觉哲学：**卖点直给，价值一眼可见**。

## Type DNA

- 节奏：商业广告节奏，快切但不碎，每个镜头都有明确的信息任务
- 视角：第三人称产品视角，观众 = 潜在用户
- 核心张力：问题/需求 -> 产品能力展示 -> 价值确认
- 禁区：不要叙事弧线，不要人物情感，不要抽象概念

## Purpose 序列

### 短视频 (<= 15s)

| 顺序 | Purpose | 时长占比 | 信息任务 |
|------|---------|----------|----------|
| 1 | hook | 2-3s | 产品结果先行，最强卖点一帧定格 |
| 2 | setup | 3-4s | 产品主界面/核心场景，建立认知框架 |
| 3 | turn | 3-4s | 关键操作或能力展示，动效打开功能 |
| 4 | payoff | 3-4s | 价值确认，成果面板/对比/数据 |
| 5 | close | 2-3s | 品牌定格或行动召唤姿态 |

### 长视频 (16-60s)

用三阶段结构，但每阶段内部保持 product-demo 节奏：
- world-build: 产品使用场景建立，用户痛点可视化
- change-push: 产品能力连续展示，功能卡片/面板依次打开
- emotional-close: 使用后的状态变化，价值确认定格

## 视觉词汇

### 画面特征

- 产品 UI 占画面 60%+，信息层级清晰可读
- 结果卡片/数据面板以浮动层或卡片形式呈现
- 动效克制但精准：滑入、展开、缩放、状态切换
- 背景干净，不抢产品视觉焦点
- 每个镜头只有一个视觉重点

### 光线与材质

- 产品摄影级布光：主光 + 补光 + 轮廓光
- UI 元素有微妙的材质感（毛玻璃、金属边、哑光面）
- 避免平面化，保持层次深度

### 色彩策略

- 主色从美学预设取
- 强调色仅用于：关键数据、完成状态、行动按钮
- 背景色不超两种，保持干净

## 音频设计

| Purpose | 声音特征 | 具体参考 |
|---------|----------|----------|
| hook | 一声抓人的产品音或确认音 | 清脆的 ding/whoosh/点击音 |
| setup | 轻量科技感环境音 | 低频 pad + 细微数据流声 |
| turn | 操作反馈音密集，节奏加快 | 连续点击音 + 界面切换音 |
| payoff | 确认击中音，配乐最高点 | 满足感强的 chime/success 音 |
| close | 干净收束，留品牌感余韵 | 短促 tail + 呼吸感静音 |

## 相机倾向

product-demo 的相机比通用模式更"商业"：

- hook: 更近 (Z3-Z4)，更快推近，第一眼就锁产品
- setup: 稳定中景 (Z5)，正面或侧 45，给产品足够画面
- turn: 跟随操作方向微移，不抢 UI 焦点
- payoff: 近景 (Z3)，85mm 压缩空间，让成果充满画面
- close: 缓慢拉远 (Z6-Z7)，产品缩小但仍是焦点

### hook (0-3s)

```
商业产品演示视频，强调卖点直给、UI清晰、动作利落。

镜头：快速推近至近景，35mm，shallow 景深，第一眼锁定产品价值。
动作：产品从模糊到清晰浮现，卖点卡片以克制动画就位。
声音：清脆确认音，立刻建立商业节奏感。
额外要求：商业广告节奏，卖点一眼可见，界面与动效高级干净。
```

### payoff (9-12s)

```
商业产品演示视频。风格锁定同上。
9-12秒画面：成果面板以 [预设网格/布局] 排列，核心数据居中，[预设材质] 质感，
强调色标记完成状态。背景保持 [预设色板] 干净衬托。
镜头：由中景推到近景聚焦中央面板，85mm，shallow 景深，让成果充满画面。
动作：数据从加载切换为完成状态，颜色变化克制但明确。
声音：满足感确认音，配乐进入最高点但保持低音量，让画面说话。
额外要求：商业广告节奏，卖点一眼可见，界面与动效高级干净。
```

## 常见错误

- hook 不要从 logo 或品牌名开始 -- 先给产品结果
- setup 不要堆叠信息 -- 一次只展示一个层级
- turn 不要用抽象动画 -- 展示真实操作或功能
- payoff 不要谦虚 -- 这是价值确认的高光时刻
- close 不要拖沓 -- 2-3 秒足够，品牌定格即走
