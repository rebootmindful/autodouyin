# Seedance Duration Boundaries

## Hard Capability Boundary

- 单段生成能力：官方与参考资料一致指向 `4-15s` 单次生成上限。
- 多模态参考仍受同一单段上限约束。

## Official Recommendation Signal

当前 Dreamina 官方页面在操作步骤里给出的时长设置是 `5s-12s`。

这说明：

1. 平台能力上限是 `15s`
2. 官方当前更鼓励用户把常规生成控制在 `5-12s`

## Practical Speed Rule

严格讲，**越短越快**。

但从“速度 + 画面信息密度 + 迭代成本”三者平衡看，建议分三档：

1. `4-5s`
   - 最快试片窗口
   - 适合测试动作、镜头、口型、转场方向
2. `5-8s`
   - 最快的实用窗口
   - 适合先验证一个镜头块是否成立
3. `8-12s`
   - 最稳的表达窗口
   - 适合单段真正承载完整信息

> 判断：`5-8s` 是最快的**实用**窗口；`4-5s` 是最快的**纯试片**窗口。  
> 这里“实用/试片”是工程判断，不是官方口号。

## Long Video Boundary

建议按执行层，不按叙事层，设置边界：

- `<=15s`
  - 单段生成
- `16-30s`
  - 2 段优先
- `31-45s`
  - 3 段优先
- `46-60s`
  - 4 段优先
- `>60s`
  - 不建议继续当作一条连续延长链
  - 应拆成独立 scene，再后期拼接

## Why >60s Changes Strategy

超过 60 秒后，连续延长链常见问题会迅速放大：

1. 风格漂移
2. 角色变脸
3. 场景关系断裂
4. 音画连续性变脆

所以 `>60s` 的正确方法不是“继续往后续”，而是：

- 先拆 scene
- scene 内部继续保持 `<=30-45s`
- scene 之间靠后期剪辑和声音桥接

## Execution Recommendation

### 想最快确认思路

先跑 `4-5s`

### 想最快确认可用片段

先跑 `5-8s`

### 想正式做单段表达

优先 `8-12s`

### 想榨满单段叙事

再上 `12-15s`

不要一开始就全上 `15s`。那是最贵的试错方式。
